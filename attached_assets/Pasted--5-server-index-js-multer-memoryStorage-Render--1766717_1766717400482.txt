### 5. server/index.js (핵심 구현)

**중요 포인트:**
- `multer.memoryStorage()` 사용 (Render 임시 파일시스템 이슈 방지)
- 프로덕션에서 `client/dist` 정적 파일 서빙
- PORT는 `process.env.PORT || 10000`
- `0.0.0.0` 바인딩 필수
- CORS 설정
- Health check 엔드포인트 (`/api/health`)

**API 엔드포인트:**
1. `POST /api/sheets` - 업로드된 파일의 시트 목록 반환
2. `POST /api/convert` - 파일 변환 및 다운로드

**ExcelJS로 구현해야 할 것:**
- Buffer에서 직접 파싱 (`workbook.xlsx.load(buffer)`)
- 출력 파일 생성 시:
  - 헤더 행 배경색 노란색 (#FFFF00)
  - 모든 셀 테두리
  - 숫자 천단위 구분 쉼표
  - 자동 너비 조정

## ⚠️ Render 배포 시 주의사항 (반드시 구현)

### 1. 메모리 효율
- multer는 반드시 `memoryStorage()` 사용
- 대용량 파일 처리 시 스트리밍 고려

### 2. 파일 시스템
- Render는 ephemeral filesystem (재시작 시 삭제)
- 디스크 저장 절대 금지, Buffer만 사용

### 3. 빌드 최적화
- `npm run build`로 client/dist 생성 확인
- Express가 `dist` 폴더 정확히 서빙하는지 확인

### 4. 환경변수
- `NODE_ENV=production` 체크
- `process.env.PORT` 사용

### 5. 헬스체크
- `/api/health` 엔드포인트로 Render 웜업 방지
